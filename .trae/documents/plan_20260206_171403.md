# CPU占用过高问题优化计划

## 问题分析

经过代码分析，发现CPU占用过高的主要原因是：

1. **计时器执行频率过高**：纪念日计算计时器每秒执行一次
2. **重复创建重量级对象**：每次执行都创建新的 `DateFormatter` 实例
3. **频繁更新UI**：每秒更新多个标签的显示内容
4. **不必要的计算**：每次执行都重新计算所有纪念日信息，即使变化不大

## 优化方案

### 1. 优化计时器执行频率

**修改点：** `startAnniversaryTimer()` 方法

**具体措施：**
- 将计时器执行频率从每秒1次降低到每5秒1次
- 对于秒级显示，使用缓存机制减少计算

### 2. 缓存重量级对象

**修改点：** `updateAnniversaryDetails()` 方法

**具体措施：**
- 将 `DateFormatter` 实例缓存为静态属性
- 避免每次都创建新的格式化器

### 3. 优化UI更新逻辑

**修改点：** `updateAnniversaryDetails()` 方法

**具体措施：**
- 只在内容真正变化时才更新UI
- 对于不常变化的信息（如下一个100天、180天等），使用缓存
- 减少富文本字符串的创建频率

### 4. 优化纪念日计算逻辑

**修改点：** `updateAnniversaryDetails()` 方法

**具体措施：**
- 缓存计算结果，只在必要时重新计算
- 对于秒级显示，使用简单的累加而非重新计算
- 优化日期计算算法，减少不必要的操作

## 技术实现细节

### 1. 优化计时器执行频率

```swift
private func startAnniversaryTimer() {
    stopAnniversaryTimer()
    updateAnniversaryDetails()
    anniversaryTimer = Timer.scheduledTimer(
        timeInterval: 5.0,  // 改为每5秒执行一次
        target: self,
        selector: #selector(updateAnniversaryDetails),
        userInfo: nil,
        repeats: true
    )
    RunLoop.current.add(anniversaryTimer!, forMode: .common)
}
```

### 2. 缓存重量级对象

```swift
// 在类级别添加静态属性
private static let dateFormatter: DateFormatter = {
    let formatter = DateFormatter()
    formatter.dateFormat = "yyyy-MM-dd"
    return formatter
}()

// 在方法中使用缓存的格式化器
func formatDate(_ date: Date) -> String {
    return Self.dateFormatter.string(from: date)
}
```

## 预期效果

### 1. CPU占用降低
